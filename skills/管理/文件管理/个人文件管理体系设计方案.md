# 个人文件管理体系 v4.3

> **版本**：4.3  
> **设计时间**：2026-02-21  
> **核心原则**：想写的放知识库，做的放AIGC/自媒体

---

## 一、核心理念

### 1.1 闭环逻辑

```
学习 ←→ 工作（相互促进）
  ↓         ↓
  └──→ 自媒体 ←┘
         ↓
    加深理解（回到学习/工作）
```

- 学习是为了更好地工作
- 工作和学习作为内容，促进自媒体
- 自媒体的增长又加深对学习/工作的理解

### 1.2 分类原则

| 类型  | 放哪       | 说明                                   |
| --- | -------- | ------------------------------------ |
| 想写的 | 知识库      | 学习笔记、复盘、想法、文章                        |
| 做的  | AIGC/自媒体 | 需求文档、代码、视频 |
| 杂物  | 仓库       | 软件安装包、过时项目                           |

**原则：桌面是临时工作台，处理完的文件归位。**

## 三、各区域职责

| 区域       | 存什么            | 不存什么    |
| -------- | -------------- | ------- |
| **马哲**   | 指导思想、方法论书籍     | 具体技术笔记  |
| **知识库**  | 日记、笔记          | 代码、视频   |
| **AIGC** | 代码项目、项目想法      | 学习笔记、复盘 |
| **自媒体**  | 视频相关（素材/草稿/成品） | 代码      |

**想法分类：**

- 模糊想法 → 知识库/日记/每日/ 或 知识库/笔记/
- 明确想做的项目想法 → AIGC/[业务]/想法/

## 六、闭环流转

**完整工作流：**

```
知识库（规划/X）
     ↓
     目标 → 计划
     ↓
AIGC/自媒体（执行/生产）
     ↓
     身体力行 → 产出代码/视频
     ↓
知识库（产出/Y）
     ↓
     总结 → 文章 → 反馈
```

| 阶段 | 位置 | 内容 |
|-----|------|------|
| 起点 | 知识库/日记 | 目标、规划（X） |
| 执行 | AIGC/自媒体 | 身体力行、生产过程 |
| 终点 | 知识库/笔记 | 产出、总结（Y） |

**核心理解：**
- 知识库 = X → Y 的载体（起点也是终点）
- AIGC/自媒体 = 执行器（中间的生产过程）

**注：仓库是杂物箱，不参与闭环流转。**

## 七、设计决策

| 决策点    | 选择                 | 理由            |
| ------ | ------------------ | ------------- |
| 5个文件夹  | 马哲、知识库、AIGC、自媒体、仓库 | 按业务分，不按类型分    |
| 知识库结构  | 日记（按时间）+ 笔记（按主题）   | 日记记录成长，笔记沉淀知识 |
| 日记归档   | 每日→周→月→季→年滚动归档     | 自动整理，不堆积      |
| AIGC结构 | 想法 + work + 已完成    | 覆盖项目全生命周期     |
| 自媒体结构  | 素材→草稿→成品           | 符合视频制作流程      |
| 仓库     | 杂物箱                | 不确定要不要的东西     |

## 八、维护规则

### 8.1 GitHub同步



**原理：**

- 本地 commit = 历史记录（可追溯、可恢复）

- 远程 push = 备份（防止本地丢失）



**恢复方式：** 如果本地记录丢失，可以从远程拉取恢复：

```

git clone https://github.com/your-username/your-repo.git

```



**同步指令：**







1. 获取本地当前分支名



2. 获取远程主分支名



3. 如果本地分支名 ≠ 远程主分支名 → 本地分支改名为远程主分支名



4. 设置本地分支跟踪远程同名分支



5. 提交更改（git add -A → git commit）



6. 推送本地当前分支到远程同名分支



7. 不动其他分支



**命令：**



```



# 获取本地当前分支名



git branch --show-current







# 获取远程主分支名



git remote show origin | grep "HEAD branch"







# 如果本地分支名 ≠ 远程主分支名，改名



git branch -m 当前分支名 远程主分支名







# 设置跟踪关系



git branch --set-upstream-to=origin/远程主分支名 远程主分支名







# 提交更改



git add -A



git commit -m "描述更改内容"







# 推送当前分支



git push origin 当前分支名



```

### 8.2 会话启动检测

**触发时机：** 每次用户与AI开始对话时

**指令：**

1. 获取系统日期和时间
2. 检查知识库/日记/每日/今天是否已有计划文件
3. 根据时间和状态，主动询问用户

**检测规则：**

| 时间段 | 今日计划状态 | AI行为 |
|-------|------------|-------|
| 5:00-12:00 | 无计划 | 主动问："早上好！今天要写计划吗？" |
| 5:00-12:00 | 有计划 | "早上好！今天计划已写，继续加油！" |
| 12:00-18:00 | 无计划 | "下午好！今天还没写计划，要补上吗？" |
| 18:00-24:00 | 无复盘 | 主动问："晚上好！今天要写复盘吗？" |
| 18:00-24:00 | 有复盘 | "晚安！今天已完成复盘，早点休息。" |

**命令：**
```
# PowerShell获取日期时间
Get-Date -Format "yyyy-MM-dd HH:mm"
```

### 8.3 每日启动

**触发方式：** 用户说"早上好"

**指令：**

1. 获取系统日期（年月日+星期）
2. 检查知识库/日记/每日/是否有昨天的文件
3. 有 → 读取昨天计划，询问哪些继续
4. 无 → 直接开始今天的计划
5. AI询问：今天最重要的3件事是什么？
6. 生成今日计划 → 用户确认 → 保存到知识库/日记/每日/

**命令：**
```
# Windows获取日期
date /t

# 或 PowerShell
Get-Date -Format "yyyy-MM-dd dddd"
```

**今日计划模板：**
```
# YYYY-MM-DD 星期X

## 今日目标（最多3件）
- [ ] 任务1
- [ ] 任务2
- [ ] 任务3

## 复盘
- 完成：
- 未完成：
- 感悟：
```

### 8.4 每日复盘

**触发方式：** 用户说"晚安"

**指令：**

1. 获取系统日期
2. 读取知识库/日记/每日/今日的计划文件
3. AI询问：今天完成了什么？有什么感悟？
4. 更新复盘内容 → 用户确认 → 保存
5. 推送到GitHub备份

**复盘问题：**
- 今天完成了哪些？
- 哪些没完成？为什么？
- 有什么感悟？

### 8.5 周复盘

**触发方式：** 用户说"周总结"

**一周定义：** 按日期跨度计算，周一到周日为一周。可以在任意一天做周总结。

**指令：**

1. 获取系统日期，确定本周范围（周一到周日）
2. 读取知识库/日记/每日/本周所有文件
3. 统计本周完成率
4. AI询问：本周有什么收获？下周有什么计划？
5. 生成周总结 → 用户确认 → 保存到知识库/日记/周/
6. 推送到GitHub备份

**周总结模板：**
```
# YYYY年第WW周总结

## 本周完成情况
- 完成：X件
- 未完成：Y件
- 完成率：Z%

## 本周收获
- 

## 下周计划
- [ ] 
- [ ] 
- [ ] 
```

### 8.6 文件整理流程

**触发方式：** 用户说"你好"

**核心理念：** 安全第一，确保每个文件都能找到

**完整流程：**

```
用户说"你好"
    ↓
扫描未被git跟踪的文件
    ↓
有未追踪的文件？
    ↓
否 → 直接开始整理文件
    ↓
是 → 检查是否已配置GitHub仓库
        ↓
    否（初次使用）→ 提醒用户提供GitHub仓库地址，帮助初始化
        ↓
    是 → 备份（git add → commit → push）
    ↓
展示未追踪文件给用户确认
    ↓
开始整理文件
    ↓
整理完成 → 推送到GitHub
```

**扫描未跟踪文件：** AI列出未被git跟踪的文件，让用户确认是否需要纳入版本控制。

**初次使用：**

**前提：**

1. 已安装git
2. 已注册GitHub账号
3. 已创建远程仓库

**需要提供：**
| 信息 | 示例 |
|-----|------|
| GitHub用户名 | your-username |
| GitHub邮箱 | your@email.com |
| 仓库地址 | https://github.com/your-username/your-repo.git |

用户提供以上信息后，AI帮助初始化本地仓库。

**交互流程：**

```
AI扫描文件 → 分批次展示 → 用户确认 → 执行 → 推送GitHub → 下一批
```

**每批数量：**
| 文件总数 | 每批数量 |
|---------|---------|
| ≤20个 | 5个/批 |
| 21-100个 | 10个/批 |
| >100个 | 20个/批 |

**AI限制：** 

- 忽略快捷方式（.lnk文件）
- 忽略Git相关配置文件（.gitignore、.gitattributes等）
- 忽略中空地带（见8.8）

**展示格式：**

```
第N批/共M批 | 已处理X个/共Y个 | 进度Z%

┌─────────────────┬───────────────┐
│ 文件            │ 目标位置      │
├─────────────────┼───────────────┤
│ card.zip        │ 仓库/         │
│ card新.zip      │ 仓库/         │
│ r任务.txt       │ 知识库/日记/每日/ │
└─────────────────┴───────────────┘

┌──────┬──────────┐
│ 回复 │ 效果     │
├──────┼──────────┤
│ 确认 │ 确认本批 │
│ 跳过 │ 跳过本批 │
└──────┴──────────┘

修改目标位置示例：仓库 → 自媒体/素材/
```

**跳过的文件：** 所有批次结束后，单独展示一批：

```
整理完成！还有X个文件被跳过：

┌─────────────────┬───────────────┐
│ 文件            │ 目标位置      │
├─────────────────┼───────────────┤
│ 文件A.zip       │ 仓库/         │
│ 文件B.txt       │ 知识库/日记/每日/ │
└─────────────────┴───────────────┘

┌──────┬──────────────┐
│ 回复 │ 效果         │
├──────┼──────────────┤
│ 确认 │ 确认本批     │
│ 跳过 │ 保留在桌面   │
│ 中空 │ 加入中空地带 │
└──────┴──────────────┘
```

**加入中空地带后：** 文件会被记录在8.4节的中空地带列表中，AI以后不再处理。

**安全原则：**

1. 每个文件都经过用户确认
2. 用户知道文件放哪了
3. 不确定的可以跳过

### 8.7 自动归档

**日记结构：**

```
日记/
├── 每日/   ← 每天写
├── 周/     ← 每周归档
├── 月/     ← 每月归档
├── 季/     ← 每季归档
└── 年/     ← 每年归档
```

**规划层级：**

```
年 → 战略层（看清时代方向，选定维度）
  ↓
季 → 战役层（这季度深耕哪个维度）
  ↓
周 → 战术层（本周具体行动）
  ↓
日 → 执行层（每天三件事）
```

**核心逻辑：日积月累，从动作到体系，应对时代变化。**

**触发时机：** AI每次会话开始时检查

**归档规则：**

1. 扫描知识库/日记/每日/里的文件
2. 读取文件日期
3. 满7天 → 归档到周/
4. 满4周 → 归档到月/
5. 满3月 → 归档到季/
6. 满4季 → 归档到年/

**识别方式：** 文件名或内容中的日期

### 8.8 中空地带

**定义：** 桌面上的某些文件夹/文件，AI识别后跳过，不参与文件整理流程

**当前中空地带：**

- 海拉鲁app/（参考项目，长期使用）
- card.zip（压缩包文件）
- card新.zip（压缩包文件）
- r任务.txt（任务记录文件）
- 个人文件管理体系设计方案.md（体系文档本身）
- 开发rules.txt（开发规则文档）
- 追忆.txt（回忆记录文件）

**添加方式：** 用户可自行在文档中添加需要忽略的区域

**规则：** 中空地带的文件不会被AI移动或删除
