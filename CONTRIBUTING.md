# Skills 仓库规范

> 本文件记录仓库维护规范，供自己和 AI 助手参考。

---

## 目录结构

```
skills/
├── README.md          # 入口索引，精简，状态前置
├── ROADMAP.md         # 规划详情，未来计划
├── CHANGELOG.md       # 版本记录
├── CONTRIBUTING.md    # 本文件，维护规范
├── LICENSE
│
├── 管理/              # "我要理一理"
├── 创作/              # "我要写/做东西"
├── 自动化/            # "我要省事"
└── 元能力/            # "我要变强"
```

---

## 分类原则

按**动作意图**分类，用户想用时第一反应知道去哪找。

| 分类 | 心智 | 示例 |
|------|------|------|
| 管理 | 整理、规划、决策 | 文件管理、决策框架 |
| 创作 | 产出内容 | 项目门面、自媒体脚本 |
| 自动化 | 自动执行 | 网关、同步、发布 |
| 元能力 | 增强能力 | 技能生成、Prompt优化 |

---

## Skill 目录规范

```
skill-name/
├── README.md     # 必须：说明文档
├── skill.md      # 必须：核心skill文件（用户发给AI）
├── design.md     # 可选：设计文档
└── assets/       # 可选：配套资源
```

---

## 新增 Skill 流程

1. **确认分类**：属于管理/创作/自动化/元能力哪个？
2. **创建目录**：在对应分类下创建 `skill-name/`
3. **编写文件**：至少包含 `skill.md` 和 `README.md`
4. **更新索引**：
   - README.md：添加到索引表
   - ROADMAP.md：从待开发移到已完成
   - CHANGELOG.md：记录版本和日期

## 状态标识

| 标识 | 含义 | 使用条件 |
|:----:|------|---------|
| ✅ | 可直接使用 | skill.md 已完成，可正常使用 |
| 🚧 | 开发中 | 有目录和部分文件，但未完成 |
| 📋 | 规划中 | 只在 ROADMAP 记录，不建目录 |

**原则**：没做好不建目录，不让用户点进去落空。

---

## 文件职责

| 文件 | 内容 | 更新时机 |
|------|------|---------|
| README.md | 精简索引，状态前置 | 新增/状态变更时 |
| ROADMAP.md | 规划详情，历史记录 | 新增/状态变更时 |
| CHANGELOG.md | 版本记录，技术细节 | 发布新版本时 |

## 仓库协作规则

`Skills-Walker` 是实验区、母仓库和孵化池。

默认协作顺序：

1. 新改动先进入 `Skills-Walker`
2. 先在实验区审理、验证、打磨
3. 确认稳定后，再同步到对应主题仓库

原则：

- 实验区优先，主题仓库后置
- 未验证的能力不直接进入主题仓库主线
- 主题仓库只承接已经相对稳定、适合对外展示的内容
