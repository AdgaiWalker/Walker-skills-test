# README 生成器

用户转化导向的 README 生成工具。

## 前提

调用此 skill 时，AI 已有充分上下文：
- 已读取项目代码/文档
- 已理解产品想法
- 已了解用户意图

## 模块触发识别

| 触发词 | 执行模块 |
|--------|----------|
| 生成 README / 写 README | 完整流程 |
| 美化 README | 只执行模块4 |
| 优化 README | 模块3+4 |

---

## 模块1：项目分析

基于已有上下文，识别项目类型：

**代码类**：库/框架、CLI 工具、网站/应用

**内容类**：Skills、理论框架、文档/教程

---

## 模块2：用户决策引导

如有缺失的关键决策点，用选项化问题引导：

**目标用户**
```
A. 开发者（需要代码示例、API）
B. 技术决策者（需要架构、对比）
C. 普通用户（需要截图、步骤）
D. AI 用户（需要触发词、示例）
E. 思考者（需要理念、方法论）
```

**核心卖点**（一句话描述）

**调性风格**
```
A. 极简实用
B. 友好亲切
C. 专业严谨
D. 理念深度
```

**美化风格**
```
A. 极简（无徽章或仅 License，纯文本为主）
B. 标准（徽章、分隔线、对齐、视觉层次）
```

如上下文已充分，直接跳过。

## 模块4：文档美化

根据用户选择的美化风格执行。

### 处理已有 README

若用户已有 README 内容：
1. 保留核心信息
2. 调整结构和层级
3. 应用美化风格
4. 补充缺失元素

### 极简美化

适用：快速出文档、项目早期、偏好简洁

特点：
- 无徽章或仅 License
- 无分隔线装饰
- 表格紧凑
- 纯文本为主
- 无对齐控制

### 标准美化

适用：正式发布、展示专业性、偏好精致

特点：
- 多徽章行（3-5 个）
- 分隔线装饰
- 表格对齐（左对齐或居中）
- 引用块突出重点
- 视觉层次分明

徽章优先级：
1. License（必选）
2. 版本号
3. 构建状态
4. 下载量

### 视觉元素

- 表格展示功能/参数
- 代码块标明语言
- 引用块突出重点

### 链接设计

- 徽章可点击
- 正文用相对路径
- 底部统一外部链接

**单模块**：只输出对应模块结果

**格式**：Markdown，可直接复制使用
